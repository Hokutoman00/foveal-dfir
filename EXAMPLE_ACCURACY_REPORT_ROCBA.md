## Downgrades by enforcement layer

- **`structural_staging`** (`< 2` independent corroborating sources): **16 / 16** — all 16 downgrades.
  - All 16 are `malfind` findings (suspected code injection). Each finding carries one independent source (the `windows.malfind` plugin row). `CONFIRMED` requires at least two independent corroborating sources actually present in the finding's evidence list; one is not enough, and the structural ceiling caps to `INDICATED`.
- **`blind_grader`**: 0 (grader disabled in this run; `--no-grader`).
- **`quarantine`**: 0 (no instruction-like spans detected inside the artifact content of any finding).

> Headline result: every `CONFIRMED` claim emitted by the heuristic investigator survived in the report only as `INDICATED`, because none of them carries enough evidence under the rule that is enforced *in code* (not in docs). This is exactly the rule the strongest competitor's `README` states and its `promotion.py` does not enforce; we enforce it.

## Missed-artifact accounting (boundary register)

For this baseline run, no items landed in `UNINSPECTED` or `LOW_CONFIDENCE_BOUNDARY` (all 4,818 findings reached at least `INDICATED`), and no areas were explicitly declared unexamined. With the grader enabled, the boundary register would carry the `LOW_CONFIDENCE_BOUNDARY` entries from `DISAGREE` cases.

The structural decision that *would* have been silently lost without the boundary register: 16 `CONFIRMED` claims becoming `INDICATED` is reported *as the downgrade*, not as a silent demotion.

## Reproduce this run

```bash
# In WSL Ubuntu, where the dump lives:
cd /var/findevil/cases/rocba/Rocba-Memory
for plugin in pslist cmdline netscan malfind; do
  sudo vol -r json -f Rocba-Memory.raw windows.$plugin > findings/$plugin.json
done

# In the repo (Windows or WSL):
python -m cases.run_rocba \
  --findings-dir /path/to/findings \
  --source-file Rocba-Memory.raw \
  --no-grader \
  --out-dir cases_outputs/rocba

# Inspect:
cat cases_outputs/rocba/summary.json
```

`audit_log.json` (~9 MB) contains the full per-finding attribution: every observer's claim, the consensus verdict, dissenters and contributors. `boundary_register.json` lists what was not resolved (empty on this baseline run).

## What this run does NOT yet show

This baseline establishes that the enforcement pipeline runs end-to-end on a real 18 GB memory image and that the structural rule (CONFIRMED requires ≥ 2 sources, counted in code) downgrades over-claims correctly. The pillars NOT yet exercised on this case:

- **Blind grader / divergence**: requires Ollama + `qwen2.5:7b` for an independent second observer. Coming next.
- **Stereo fusion**: needs grader-annotated verdicts to reconstruct the kill-chain across two views.
- **Actor cadence + falsifier**: needs the disk image (`rocba-cdrive.e01`) for event timelines and case-specific hypothesis killers.
- **Prior-fit anomaly**: needs the disk image for registry / log artifacts to test against a "normal Windows host" prior.

These follow the disk image download completing and Ollama coming online.
