## Downgrades by enforcement layer

- **`structural_staging`** (`< 2` independent corroborating sources): **16 / 16** — all 16 downgrades.
  - All 16 are `malfind` findings (suspected code injection). Each finding carries one independent source (the `windows.malfind` plugin row). `CONFIRMED` requires at least two independent corroborating sources actually present in the finding's evidence list; one is not enough, and the structural ceiling caps to `INDICATED`.
- **`blind_grader`**: 0 (grader disabled in this run; `--no-grader`).
- **`quarantine`**: 0 (no instruction-like spans detected inside the artifact content of any finding).

> Headline result: every `CONFIRMED` claim emitted by the heuristic investigator survived in the report only as `INDICATED`, because none of them carries enough evidence under the rule that is enforced *in code* (not in docs). This is exactly the rule the strongest competitor's `README` states and its `promotion.py` does not enforce; we enforce it.

## Reproduce this run

```bash

cd /var/findevil/cases/rocba/Rocba-Memory
for plugin in pslist cmdline netscan malfind; do
  sudo vol -r json -f Rocba-Memory.raw windows.$plugin > findings/$plugin.json
done

# In the repo (Windows or WSL):
python -m cases.run_rocba \
  --findings-dir /path/to/findings \
  --source-file Rocba-Memory.raw \
  --no-grader \
  --out-dir cases_outputs/rocba

# Inspect:
cat cases_outputs/rocba/summary.json
```

### What this slice of the audit log shows (the sharp result)

For all 16 findings, **the investigator AND the independent grader both judged `CONFIRMED`** — they agreed, with no anchoring, that the evidence supports the claim. And yet the consensus verdict is `INDICATED`, not `CONFIRMED`, because the evidence structure carries only **one** independent source per finding, and the rule — code-enforced in `verifier/staging.py` — requires two:

> **The pipeline holds the line at `INDICATED` even when both observers want to relax to `CONFIRMED`.**

This is the IR-Accuracy / Constraint-Implementation argument in its sharpest form: the rule everyone writes in their README (CONFIRMED requires ≥ 2 independent corroborating sources) does not become a real constraint until it is enforced in code, *over the consensus of the observers themselves*. We enforce it.

### Divergence is intentionally not exercised here

A `DISAGREE` escalate-set entry would require the investigator and the grader to land on opposite sides of the INDICATED threshold. On these findings, the investigator's interpretation (`"Memory region with executable permissions and no backing file -- candidate injected code"`) stays close to what the evidence directly shows, and the grader independently arrives at the same direction. That is the *correct* outcome on this slice; manufacturing a DISAGREE by over-interpreting would be dishonest.

The synthetic toy samples (`samples/findings.json`, exercised by `run_prototype.py`) include cases designed to *force* `DISAGREE` — `F-002` (single source, investigator over-claims `CONFIRMED`, grader judges `INFERRED`) and `F-004` (over-claim caught by grader). Those scenes are where the rescue / escalate beat shows; on the Rocba memory slice, the dominant beat is the **structural-rule-overrides-observer-consensus** result above.
